# Project: Опекун (Opekun)

Интеллектуальный Telegram-бот — «AI-няня» и заботливый наставник. Бот совмещает роль любящей матери и опытного тьютора, помогая пользователю в обучении, планировании дня и заботе о себе.

## Технологический стек

- **Core**: Node.js, TypeScript
- **Telegram API**: [grammY](https://grammy.dev/) (Long polling)
- **AI/LLM**: [Vercel AI SDK](https://sdk.vercel.ai/) (интеграция через OpenRouter)
- **Database**: PostgreSQL (Neon) + [Drizzle ORM](https://orm.drizzle.team/)
- **Background Jobs / Scheduler**: [BullMQ](https://docs.bullmq.io/) + Redis (Upstash)
- **Memory**: [Mem0](https://mem0.ai/) (семантическая долгосрочная память)
- **Integrations**: MCP (Model Context Protocol) для внешних инструментов (Google Calendar и др.)
- **Logging**: [Pino](https://getpino.io/)
- **Testing**: [Vitest](https://vitest.dev/)

## Архитектура проекта

Проект разделен на несколько функциональных слоев:

- `src/agent/`: Мозги бота. Содержит оркестратор (`orchestrator.ts`), системные промпты и инструменты (tools) для AI.
- `src/bot/`: Слой Telegram. Обработчики команд, текстовых и голосовых сообщений, а также middleware (авторизация, контекст, лимиты).
- `src/db/`: Работа с данными. Определение схемы (`schema.ts`) и репозитории для доступа к данным (`repos/`).
- `src/scheduler/`: Управление очередями задач. Фоновые воркеры и логика проактивных уведомлений.
- `src/mcp/`: Клиент для протокола MCP, позволяющий агенту взаимодействовать с внешними сервисами.
- `src/voice/`: Обработка голоса (STT и TTS).

## Основные команды

### Разработка
- `npm run dev` — Запуск проекта в режиме разработки с автоматической перезагрузкой (`tsx watch`).
- `npm test` — Запуск тестов Vitest.

### Сборка и запуск
- `npm run build` — Компиляция TypeScript в JavaScript (`dist/`).
- `npm start` — Запуск собранного проекта из папки `dist/`.

### База данных
- `npm run db:push` — Прямое применение изменений схемы к базе данных (рекомендуется для быстрой итерации).
- `npm run db:generate` — Генерация файлов миграций.
- `npm run db:studio` — Запуск визуального интерфейса Drizzle Studio для управления данными.

## Конвенции разработки

1. **Репозитории**: Для работы с БД используйте паттерн Repository (см. `src/db/repos/`).
2. **Логирование**: Используйте `createChildLogger` из `src/lib/logger.ts` для логирования в каждом модуле.
3. **AI Инструменты**: Новые возможности для агента добавляются через создание инструментов в `src/agent/tools/`.
4. **Проактивность**: Бот может писать первым через BullMQ задачи (например, `morning_greeting`). Логика запуска находится в `src/scheduler/worker.ts`.
5. **Память**: Важные факты о пользователе должны сохраняться в Mem0 через инструмент `memory_save` или автоматически оркестратором.

## MCP Серверы
Конфигурация подключенных MCP-серверов находится в `src/mcp/servers.json`. Бот поддерживает Google Calendar и другие инструменты через этот протокол.
