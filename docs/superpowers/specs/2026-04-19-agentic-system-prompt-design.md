# Дизайн: Прокачка агентного поведения Опекуна

**Дата:** 2026-04-19  
**Подход:** Рефакторинг промпта с алгоритмом (Вариант Б)

---

## Проблема

Текущий системный промпт описывает инструменты как документацию («что делает»), а не как инструкции («когда вызывать»). Агент пассивен — ждёт явных команд, не распознаёт косвенные сигналы, не выстраивает цепочки действий самостоятельно.

---

## Изменения

### 1. Алгоритм каждого ответа

Новый раздел в начале промпта — жёсткая очерёдность для каждого сообщения пользователя:

1. **КОНТЕКСТ** — `memory_search` по ключевым словам
2. **РАСПОЗНАВАНИЕ** — проверить триггеры молчаливых действий
3. **ДЕЙСТВИЕ** — выполнить основной запрос
4. **СОХРАНЕНИЕ** — `memory_save` если узнал новый факт
5. **ОТВЕТ** — `message_send_text` или `message_send_voice`
6. **ПОДТВЕРЖДЕНИЕ** — курсивом в конце если были молчаливые действия

**Молчаливые действия с лёгким подтверждением:**
- Расход распознан → `expense_add` автоматически → `_(записал: 500₸, еда)_`
- Факт о пользователе → `memory_save` без упоминания
- Задача с дедлайном → `todo_add` автоматически → `_(добавил в список дел)_`

### 2. Расширенные триггеры тулов

**expense_add** — вызывать при любом упоминании денег:
- Прямые: "потратил", "заплатил", "купил", "стоит X тенге/рублей"
- Косвенные: "сходил поел", "взял кофе", "заказал доставку", "взял такси", "сходили в кино", "оплатил", "закинул"
- Фото чека → всегда `expense_add`

**todo_add** — вызывать при:
- Прямые: "не забыть", "напомни", "надо сделать", "добавь в список"
- Косвенные: "завтра нужно...", "кстати надо бы...", "хочу не забыть..."
- Если есть временной маркер → deadline обязателен

**memory_save** — вызывать при любом новом факте:
- Личное: семья, друзья, работа, учёба, здоровье
- Предпочтения: еда, места, время активности
- Паттерны: регулярные траты, время отклика, проблемные привычки

### 3. Эскалация при молчании

После любого проактивного сообщения (расписание, напоминание):
- **Попытка 1:** `followup_ask(delayMinutes: 3)`
- **Попытка 2:** если задача/привычка важная → `call_user` немедленно
- **Попытка 2:** если обычное → `followup_ask(delayMinutes: 5)`, затем стоп

Важность определяется: наличием дедлайна сегодня, или явной пометкой важности в контексте задачи.

### 4. Новая цепочка morning_greeting

Обязательная последовательность (все 6 шагов):

1. `weather_get_forecast`
2. `gcal_list_all_events` (сегодня)
3. `habit_list`
4. `todo_list` (невыполненные с дедлайном сегодня + просроченные)
5. `expense_list(period: 'today')` за вчера
6. `web_search` — мотивирующий факт по интересам пользователя

Всё собирается в **одно сообщение**:
- Погода + совет по одежде
- Встречи на день
- Todo на сегодня (если есть)
- Невыполненные привычки вчера (мягко)
- Расходы вчера (если > 0)
- Факт/цитата по интересам пользователя

### 5. Паттерн-обучение

Агент наблюдает за поведением и сохраняет выводы в `memory_save` автоматически:

| Паттерн | Что сохранять | Действие |
|---------|--------------|----------|
| Не отвечает до времени X | "не беспокоить до HH:MM" | Сдвинуть followup |
| Расход в категории 3+ раз/неделю | "регулярно тратит на [категория]" | Учитывать в статистике |
| Систематически пропускает привычку | "трудности с [привычка]" | Мягко поднять тему |
| Активен в определённое время | "обычно активен в HH:MM–HH:MM" | Подстроить время сообщений |

## Проверка

После изменений проверить:
1. `npx tsc --noEmit` — компиляция без ошибок
2. `npm run build` — сборка бандла
3. Вручную в боте: написать "сходил в кафе 800 тенге" — должен тихо записать расход
4. Вручную: написать "завтра надо позвонить врачу" — должен добавить todo
5. Утреннее приветствие — проверить что включает все 6 элементов
